Explore this codebase and produce a clear overview for someone new to the project.

1. Identify the project type (web app, library, CLI tool, service, etc.)
2. Identify the main languages, frameworks, and key dependencies
3. Map the high-level architecture:
   - Entry points
   - Main modules/components
   - How code is organized (feature-based, layer-based, etc.)
4. Note any non-obvious patterns or conventions used
5. Document how to get started (setup, run, test)

Keep the output concise but comprehensive. Focus on what someone needs to understand to be productive quickly.
